包管理工具之PNPM

安装pnpm本身： 

    npm install -g pnpm

安装依赖包

命令 说明
pnpm add vislite 保存到 dependencies
pnpm add -D vislite 保存到 devDependencies
pnpm add -O vislite 保存到 optionalDependencies
pnpm add -g vislite Install package globally
pnpm add vislite@alpha 从 alpha 标签下安装
pnpm add vislite@0.8.0 安装指定版本 0.8.0

一些特殊安装方法

安装本地项目

比如我们有一个包vislite，其本地项目工程位置为： /Users/zxl20070701/Desktop/VISLite，可以直接： 

    pnpm add vislite@/Users/zxl20070701/Desktop/VISLite

下载所有依赖 

    pnpm install

版本更新

命令 说明
pnpm update 遵循 package.json 指定的范围更新所有的依赖项
pnpm update --latest 更新所有依赖项，此操作会忽略 package.json 指定的范围
pnpm update vislte@1 将 vislte 更新到 v1 上的最新版本
pnpm update "@zxl20070701/*" 更新 @zxl20070701 范围内的所有依赖项

工作空间（Workspace）

基本使用

比如我们现在有一个项目，项目中有两个子项目，大体结构如下： 

- packages
    - pkg1
        - package.json
    - pkg2
        - package.json
- package.json
- pnpm-workspace.yaml

其中pnpm-workspace.yaml内容如下： 

packages:
  - 'packages/*'

而pkg1和pkg12的项目名称分别是mypkg1和mypkg2。

主目录下安装：

比如在项目根项目下安装pkg1，只需要： 

    pnpm install mypkg1 -w

子项目下安装：

比如在子项目pkg2下安装pkg1，只需要： 

cd packages/pkg1
pnpm install mypkg1
